# Contributing to xLPA (DSA Studio)

Thank you for your interest in contributing to xLPA! We are excited to have you in our community.

## 💡 How to Contribute

There are many ways to contribute to xLPA:

- **Reporting Bugs**: Found a bug? Open an issue with a clear description and steps to reproduce.
- **Feature Requests**: Have a great idea for xLPA? Let us know!
- **Code Contributions**: Submit pull requests for bug fixes or new features.
- **Documentation**: Help us improve our guides and README.

## 🎓 Sharing Interview Experiences (Highly Recommended)

One of the most valuable ways to contribute to xLPA is by sharing your **personal interview experiences**. 

Whether it's a specific question asked in a technical round for:
- Machine Coding
- Data Structures & Algorithms
- System Design
- HR/Behavioral rounds

We welcome questions and patterns you've encountered from any module. Your experiences directly help others prepare more effectively for their "next new LPA".

## 🛠️ Development Setup

1. Fork the repository.
2. Clone your fork: `git clone https://github.com/your-username/dsa-learner.git`
3. Install dependencies: `npm install`
4. Create a new branch: `git checkout -b feature/my-new-feature`
5. Make your changes and commit them: `git commit -m 'Add some feature'`
6. Push to the branch: `git push origin feature/my-new-feature`
7. Submit a pull request.

## 📜 Pull Request Guidelines

- Ensure your code follows the project's coding standards.
- Write descriptive commit messages.
- Provide a clear summary of your changes in the pull request description.
- Test your changes thoroughly before submitting.

We appreciate your contributions!
